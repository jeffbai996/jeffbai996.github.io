## Task 1: Focus trap in Search modal

**Context:** Search.jsx renders a modal dialog (`role="dialog" aria-modal="true"`). Currently pressing Tab cycles focus through the entire page behind the overlay. A focus trap constrains Tab/Shift+Tab to elements inside the modal.

**Files:**
- Modify: `frontend/src/components/Search.jsx`

The modal has exactly these focusable elements in DOM order:
1. The text `<input>` (already `ref={inputRef}`)
2. Result `<button>` items (dynamic, may be 0)
3. The "Clear" recent-searches `<button>` (conditional)
4. The close affordance is ESC-only (no close button)

Because the result list is dynamic, the trap queries the modal container for all focusable elements at keydown time rather than storing refs upfront.

- [ ] **Step 1: Add a ref for the modal container**

In `Search.jsx`, find the existing refs block near the top of the `Search` function:

```jsx

```

Add one line below it:

```jsx

const modalRef = useRef(null)
```

- [ ] **Step 2: Attach the ref to the modal div**

Find:
```jsx
<div className="search-modal" role="dialog" aria-modal="true" aria-label="Site search" onClick={e => e.stopPropagation()}>
```

Replace with:
```jsx
<div ref={modalRef} className="search-modal" role="dialog" aria-modal="true" aria-label="Site search" onClick={e => e.stopPropagation()}>
```

- [ ] **Step 3: Add the focus-trap useEffect**

Add this `useEffect` immediately after the existing "Handle escape key" `useEffect` block (which ends around line 86). Place it before the "Search function" `useEffect`:

```jsx
// Trap focus inside the modal while open
useEffect(() => {
  if (!isOpen || !modalRef.current) return

  const FOCUSABLE = 'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'

  const handleTab = (e) => {
    if (e.key !== 'Tab') return

    const focusable = Array.from(modalRef.current.querySelectorAll(FOCUSABLE))
    if (focusable.length === 0) return

    const first = focusable[0]
    const last = focusable[focusable.length - 1]

    if (e.shiftKey) {
      // Shift+Tab: if focus is on first element, wrap to last
      if (document.activeElement === first) {
        e.preventDefault()
        last.focus()
      }
    } else {
      // Tab: if focus is on last element, wrap to first
      if (document.activeElement === last) {
        e.preventDefault()
        first.focus()
      }
    }
  }

  document.addEventListener('keydown', handleTab)
  return () => document.removeEventListener('keydown', handleTab)
}, [isOpen])
```

## Task 2: AbortController in GeminiService

**Context:** `geminiService.generateResponse()` in `frontend/src/services/geminiService.js` fires a `chat.sendMessage()` call with a 30s timeout race (added in the previous session). If the user sends a new message before the first resolves, both requests are in-flight simultaneously — the older response can land after the newer one and overwrite it in the UI.

The fix: store an `AbortController` on the class instance. Each new `generateResponse` call aborts the previous one first. ChatWidget calls `geminiService.cancel()` on unmount too, preventing setState-on-unmounted-component warnings.

Note: The Google Generative AI SDK doesn't natively accept an `AbortSignal` on `sendMessage`, so aborting is implemented by rejecting a shared promise that races alongside the SDK call — same pattern as the timeout, but cancellable.

**Files:**
- Modify: `frontend/src/services/geminiService.js`
- Modify: `frontend/src/components/ChatWidget.jsx`

- [ ] **Step 1: Add controller state to GeminiService constructor**

In `geminiService.js`, find the constructor body. After `this.initialized = false` / `this.genAI = ...` setup, add:

```js
// Tracks the in-flight request so new calls can cancel the previous one
this._abortController = null
```

Add it at the very end of the `constructor`, just before the closing `}`:

```js
constructor() {
  const apiKey = import.meta.env.VITE_GEMINI_API_KEY;

  if (!apiKey || apiKey === 'your_api_key_here') {
    this.initialized = false;
    this._abortController = null;
    return;
  }

  try {
    this.genAI = new GoogleGenerativeAI(apiKey);
    // ... existing model setup stays unchanged ...
    this._abortController = null;
  } catch (e) {
    this.initialized = false;
    this._abortController = null;
  }
}
```

Only add `this._abortController = null` in the relevant spots — don't restructure the existing constructor logic.

- [ ] **Step 2: Add the cancel() method**

Find the `getStatus()` method near the bottom of the class (around line 542). Add `cancel()` immediately before it:

```js
/**
 * Cancel any in-flight generateResponse call.
 * Safe to call even when no request is in-flight.
 */
cancel() {
  if (this._abortController) {
    this._abortController.abort()
    this._abortController = null
  }
}
```

- [ ] **Step 3: Thread AbortController through generateResponse**

In `generateResponse`, find the block that currently starts with:

```js
// Send the user's message — race against 30s timeout to prevent infinite hangs

  setTimeout(() => reject(new Error('Gemini request timed out after 30s')), 30000)
)
const result = await Promise.race([chat.sendMessage(userMessage), timeout]);
```

Replace with:

```js
// Cancel any previous in-flight request before starting a new one
this.cancel()

// Create a new controller for this request
this._abortController = new AbortController()
const { signal } = this._abortController

// Race: SDK call vs 30s timeout vs explicit cancel
const timeout = new Promise((_, reject) =>
  setTimeout(() => reject(new Error('Gemini request timed out after 30s')), 30000)
)
const aborted = new Promise((_, reject) => {
  signal.addEventListener('abort', () => reject(new Error('Request cancelled')))
})

let result
try {
  result = await Promise.race([chat.sendMessage(userMessage), timeout, aborted])
} finally {
  // Clear the controller whether we succeeded, timed out, or were cancelled
  this._abortController = null
}
```
